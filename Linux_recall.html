Commandes Terminal Essentielles

Maîtrise la ligne de commande. Voici les commandes les plus utiles avec leurs paramètres (flags) expliqués.

📂 Navigation & Manipulation de Fichiers

pwd

Affiche le chemin complet du dossier dans lequel tu te trouves actuellement (Print Working Directory).

$ pwd
/home/hakem/projets/sncf

ls

Liste les fichiers et dossiers présents dans le répertoire courant.

$ ls

Paramètres utiles :

  • -l : Format long (affiche droits, propriétaire, taille, date).
    $ ls -l
  • -a : Affiche les fichiers cachés (qui commencent par un point).
    $ ls -a
  • -h : Tailles lisibles par un humain (Ko, Mo, Go). Souvent combiné avec -l.
    $ ls -lh

cd

Change de répertoire (Change Directory).

$ cd Documents/

Paramètres utiles :

  • .. : Remonte au dossier parent (précédent).
    $ cd ..
  • ~ : Retourne direct à ton dossier personnel (/home/user).
    $ cd ~
  • - : Retourne au dernier dossier où tu étais.
    $ cd -

mkdir

Crée un ou plusieurs nouveaux dossiers.

$ mkdir dataset

Paramètres utiles :

  • -p : Crée aussi les dossiers parents s'ils n'existent pas.
    $ mkdir -p data/train/images

touch

Crée un fichier vide (ou met à jour la date de modification d'un fichier existant).

$ touch script.py

cp

Copie des fichiers ou des répertoires.

$ cp fichier.txt copie.txt

Paramètres utiles :

  • -r : Récursif (obligatoire pour copier un dossier entier).
    $ cp -r dossier1/ dossier2/
  • -i : Demande confirmation avant d'écraser un fichier existant.

mv

Déplace un fichier/dossier OU le renomme (s'il est déplacé au même endroit avec un autre nom).

# Déplacer :
$ mv data.csv ./archives/

# Renommer :
$ mv old_name.txt new_name.txt

rm

Supprime des fichiers ou dossiers. ⚠️ Attention, il n'y a pas de corbeille !

$ rm obsolete.txt

Paramètres utiles :

  • -r : Supprime un dossier et tout son contenu (Récursif).
    $ rm -r vieux_dossier/
  • -f : Force (ne demande aucune confirmation, ignore les erreurs).
  • -rf : Le duo destructeur. À utiliser avec extrême précaution.

find

Recherche des fichiers et des dossiers de manière très puissante.

# Cherche le fichier "train.py" depuis le dossier actuel (.)
$ find . -name "train.py"

Paramètres utiles :

  • -name : Cherche par nom exact (accepte les jokers *).
    $ find /home -name "*.jsonl"
  • -type : d (dossier) ou f (fichier).
    $ find . -type d -name "data"

📄 Lecture & Traitement de Texte

echo

Affiche du texte ou la valeur d'une variable. Souvent couplé au symbole > pour écrire ce texte dans un fichier.

# Affiche "Bonjour"
$ echo "Bonjour"

# Écrit "Test" dans le fichier (et l'écrase)
$ echo "Test" > info.txt

# AJOUTE "Test2" à la fin du fichier
$ echo "Test2" >> info.txt

# Affiche une variable d'environnement
$ echo $PATH

cat

Affiche tout le contenu d'un fichier directement dans le terminal.

$ cat requirements.txt

less

Affiche le contenu d'un fichier page par page (idéal pour les très longs fichiers JSON ou logs). Appuie sur `q` pour quitter.

$ less dataset_sncf.jsonl

head / tail

Affiche respectivement les 10 premières ou 10 dernières lignes d'un fichier.

$ head data.csv
$ tail data.csv

Paramètres utiles :

  • -n : Spécifier le nombre de lignes.
    $ head -n 50 fichier.txt
  • tail -f : "Follow". Lit le fichier en continu (génial pour voir un log s'écrire en direct lors d'un entraînement de modèle !).
    $ tail -f training_loss.log

grep

Recherche un mot spécifique (ou une expression régulière) à l'intérieur d'un ou plusieurs fichiers.

# Cherche le mot "Epoch" dans le fichier log
$ grep "Epoch" results.log

Paramètres utiles :

  • -i : Ignore la casse (majuscule/minuscule).
  • -r : Cherche dans tous les fichiers d'un dossier (Récursif).
    $ grep -r "import torch" .
  • -v : Inverse la recherche (affiche tout SAUF les lignes avec le mot).

wc

Compte le nombre de lignes, mots et caractères d'un fichier (Word Count).

$ wc script.py

Paramètres utiles :

  • -l : Ne compte QUE les lignes. Très utile pour savoir combien de lignes a ton dataset.
    $ wc -l dataset.jsonl

nano / vim

Éditeurs de texte directement dans le terminal. Nano est très facile pour les débutants (les raccourcis comme Ctrl+X pour quitter sont écrits en bas). Vim est très puissant mais complexe (Souviens-toi : Esc puis :q! pour fuir !).

$ nano mon_script.py
$ vim mon_script.py

⚙️ Système, Stockage & Processus

top / htop

Affiche l'utilisation en temps réel des ressources (CPU, RAM) par les processus (comme le gestionnaire des tâches Windows). htop est plus coloré et lisible (souvent à installer).

$ top
$ htop

Appuie sur 'q' pour quitter.

ps

Prend un instantané des processus en cours d'exécution.

$ ps

Paramètres utiles :

  • aux : Affiche absolument TOUS les processus tournant sur la machine, avec beaucoup de détails.
    $ ps aux | grep python

kill

Permet de forcer l'arrêt d'un programme bloqué en utilisant son numéro d'identification (PID, que l'on trouve avec `ps` ou `top`).

$ kill 1234

Paramètres utiles :

  • -9 : Signal "SIGKILL". Tue le processus immédiatement et sans pitié.
    $ kill -9 1234

df

Affiche l'espace disque disponible et utilisé sur toutes les partitions (Disk Free).

$ df -h

Le -h (human-readable) est indispensable pour voir des Go ou Mo plutôt que des blocs bruts.

du

Calcule la taille d'un dossier ou d'un fichier (Disk Usage). Utile pour trouver ce qui prend de la place.

$ du -sh /home/hakem/dataset

Paramètres utiles :

  • -s : Sommaire (donne juste le total du dossier, sans détailler chaque sous-fichier).
  • -h : Human-readable (en Mo, Go).

free

Affiche la quantité de mémoire vive (RAM) et de SWAP utilisée et libre sur le système.

$ free -h

du -sh *

Affiche les fichiers et dossiers de ce repertoire (méme les caché) et les tris par espace de stockage pris

$ du -sh * .[!.]* | sort -h
  • -s : Sommaire (donne juste le total du dossier, sans détailler chaque sous-fichier).
  • -h : Human-readable (en Mo, Go).

df

Affiche le quota de disk prix et combien il reste

$ df -h ./
  • -h : Human-readable (en Mo, Go).

🔐 Droits & Utilisateurs

sudo

"Super User DO". Permet d'exécuter une commande avec les droits d'administrateur (root). On te demandera ton mot de passe.

$ sudo apt update
$ sudo rm -rf fichier_protege/

chmod

Change les permissions d'un fichier (Lecture: r, Écriture: w, Exécution: x).

# Rend un script Python exécutable
$ chmod +x train_model.py

Paramètres utiles :

  • +x : Ajoute le droit d'exécution.
  • 755 : Mode numérique (Toi : Lire/Écrire/Exécuter. Autres : Lire/Exécuter).
    $ chmod 755 script.sh

chown

Change le propriétaire (owner) d'un fichier ou dossier (nécessite souvent `sudo`).

$ sudo chown hakem mon_fichier.txt

Paramètres utiles :

  • -R : Change le propriétaire de tout le dossier et ses sous-fichiers.
    $ sudo chown -R hakem projet/

passwd

Permet de changer ton mot de passe. Dans le terminal Linux, quand tu tapes ton mot de passe, rien ne s'affiche à l'écran (c'est normal, c'est pour la sécurité).

$ passwd

🌐 Réseau & Utilitaires Divers

ping

Envoie des paquets vers un serveur pour vérifier si tu as une connexion internet ou s'il est accessible.

$ ping google.com

Paramètres utiles :

  • -c : Nombre maximum de pings à envoyer (sinon ça ne s'arrête jamais, fais Ctrl+C pour stopper).
    $ ping -c 4 google.com

wget / curl

Deux commandes pour télécharger des fichiers depuis internet. wget télécharge le fichier, curl l'affiche dans le terminal (ou fait des requêtes API).

# Télécharge directement le fichier
$ wget https://site.com/dataset.zip

# curl affiche dans le terminal, on utilise -O pour sauvegarder
$ curl -O https://site.com/dataset.zip

tar

L'outil de base pour créer ou extraire des archives (les fameux fichiers .tar.gz).

Extraire (dézipper) :

$ tar -xzvf archive.tar.gz

Compresser un dossier :

$ tar -czvf nom_archive.tar.gz dossier/

Moyen mnémotechnique : -x pour eXtract (extraire), -c pour Create (créer).

man

Affiche le manuel complet d'une commande. C'est ta meilleure bouée de sauvetage ! (Appuie sur 'q' pour quitter).

$ man ls
$ man find

history

Affiche la liste de toutes les dernières commandes que tu as tapées dans le terminal.

$ history

# Astuce : Cherche une ancienne commande
$ history | grep "python"

L'astuce en or : Le Pipe ( | )

Le symbole "pipe" | permet de brancher la sortie d'une commande dans l'entrée d'une autre. C'est le super-pouvoir de Linux.

# Liste les fichiers (ls) ET filtre seulement ceux contenant "csv" (grep)
$ ls -l | grep ".csv"

# Compte (wc) le nombre de processus python en cours (ps)
$ ps aux | grep "python" | wc -l

🖥️ Calcul Haute Performance & SLURM

Comprendre un script SLURM (.sh)

Sur un cluster, tu ne lances pas ton code Python directement. Tu crées un "script de soumission" (souvent nommé job.sh). Ce script contient des commentaires spéciaux qui commencent par #SBATCH. SLURM lit ces commentaires pour savoir de quelles ressources tu as besoin (GPU, RAM, temps), te place dans une file d'attente, et exécute tes commandes quand les ressources sont libres. 

#!/bin/bash
# --- 1. CONFIGURATION DES RESSOURCES (Lues par SLURM) ---
#SBATCH --job-name=bert_train       # Nom de ton job (pour le retrouver)
#SBATCH --output=logs/job_%j.out    # Où écrire les print() du code (%j = ID du job)
#SBATCH --error=logs/job_%j.err     # Où écrire les messages d'erreur
#SBATCH --partition=gpu             # File d'attente à utiliser (ex: gpu, cpu)
#SBATCH --gres=gpu:1                # Je veux 1 carte graphique (GPU)
#SBATCH --time=12:00:00             # Temps max alloué (12 heures, après il coupe)
#SBATCH --mem=32G                   # Quantité de RAM système demandée

# --- 2. COMMANDES À EXÉCUTER ---
echo "Lancement du job sur le noeud: $HOSTNAME"

# Charger l'environnement ou les modules si nécessaire
module load python/3.10
source venv/bin/activate

# Lancer le vrai code
python train_model.py

sbatch

Soumet ton script `.sh` au gestionnaire SLURM. Il te retournera un numéro d'identification (le Job ID).

$ sbatch job.sh
Submitted batch job 543210

squeue

Affiche la file d'attente des jobs sur le cluster. Tu peux voir si ton code est en train de tourner (Status: `R` pour Running) ou s'il attend (Status: `PD` pour Pending).

# Voir UNIQUEMENT tes propres jobs
$ squeue -u $USER

scancel

Annule ou tue immédiatement un job que tu as soumis (utilise le Job ID donné par sbatch ou squeue).

# Annuler un job spécifique
$ scancel 543210

# Annuler TOUS tes jobs d'un coup (Attention !)
$ scancel -u $USER

sinfo

Affiche l'état global du cluster et des partitions. Permet de voir quels nœuds de calcul sont `idle` (libres), `alloc` (occupés) ou `down` (en panne).

$ sinfo
